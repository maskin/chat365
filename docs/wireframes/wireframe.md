# アプリケーション画面設計（ワイヤーフレーム）

## 1. 設計思想

- **直感的な操作:** 誰でもマニュアルなしで使えるシンプルなUIを目指す。
- **状態の可視化:** スケジュールの状態（待機中、完了など）が色やアイコンで一目でわかるようにする。
- **誤操作の防止:** 緊急放送など、クリティカルな操作には確認ステップを設ける。

---

## 2. 画面構成

### 2.1. メイン画面 (スケジュール一覧)

```
+------------------------------------------------------------------------------+
| 音声放送システム                                        [緊急放送(F1)] [＋新規放送] |
+------------------------------------------------------------------------------+
|                                                                              |
| [フィルター/検索エリア]  日付: [YYYY-MM-DD v]  ステータス: [すべて v]          |
|                                                                              |
| ---------------------------------------------------------------------------- |
|                                                                              |
| [ ] 09:00 [優先度:通常] [ステータス:待機中] 全校生徒へ。本日は通常通り授業を... [編集] [削除] |
|                                                                              |
| [ ] 12:30 [優先度:通常] [ステータス:待機中] 昼休みの連絡です。図書館の開館... [編集] [削除] |
|                                                                              |
| [x] 15:00 [優先度:緊急] [ステータス:完了]   至急連絡。3階で水漏れが発生しま... [詳細]       |
|                                                                              |
|                                                                              |
+------------------------------------------------------------------------------+
```

- **ヘッダー:** 緊急放送ボタンと新規放送ボタンを常にアクセスしやすい右上に配置。
- **一覧表示:** 時刻順にソートされた放送タスクをリスト表示。優先度やステータスはバッジで視覚的に表現。

### 2.2. 新規放送登録モーダル

```
+-------------------------------------------------+
| 新規放送の登録                                  |
+-------------------------------------------------+
|                                                 |
| [放送内容]                                      |
| +---------------------------------------------+ |
| |                                             | |
| | (ここに音声認識されたテキストが表示される)  | |
| |                                             | |
| +---------------------------------------------+ |
|                                [音声入力開始]   |
|                                                 |
| [放送日時]                                      |
| [YYYY-MM-DD] [HH:MM]                            |
|                                                 |
| [優先度]                                        |
| [通常 v] (通常, 重要)                           |
|                                                 |
|                                                 |
|                      [キャンセル] [登録する]    |
+-------------------------------------------------+
```

- **音声入力:** ボタンを押すとマイクが起動。認識されたテキストがリアルタイムで表示され、手動修正も可能。
- **シンプル設定:** MVPでは日時と優先度のみを設定項目とする。

### 2.3. 緊急放送モーダル

```
+-------------------------------------------------+
| !! 緊急放送 !!                                  |
+-------------------------------------------------+
|                                                 |
| [放送内容]                                      |
| +---------------------------------------------+ |
| | (緊急放送の内容をここに録音・表示)          | |
| +---------------------------------------------+ |
|                                [音声入力開始]   |
|                                                 |
|                                                 |
| [!] この放送は現在スケジュールされている放送に  |
|     割り込んで、即座に実行されます。            |
|                                                 |
|                      [キャンセル] [すぐに放送]  |
+-------------------------------------------------+
```

- **ワンステップ操作:** 録音して内容を確認後、「すぐに放送」ボタンを押すだけで実行される。
- **警告表示:** 割り込み放送であることを明確に伝え、誤操作を防ぐ。
